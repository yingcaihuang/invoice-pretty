# GitHub Release 截图展示配置指南

## 🎯 配置目标

在GitHub Actions自动发布的Release页面中添加项目截图展示，让用户在下载前就能看到软件的实际界面。

## 📸 添加的截图内容

### 1. 界面预览部分
在Release Notes的开头添加了界面预览：

```markdown
## 📸 界面预览

### 主界面
![系统主界面](https://raw.githubusercontent.com/yingcaihuang/invoice-pretty/main/img/assets/img1.png)

### 处理界面
![处理界面](https://raw.githubusercontent.com/yingcaihuang/invoice-pretty/main/img/assets/img2.png)
```

### 2. 功能展示部分
在技术特性后添加了详细的功能展示：

```markdown
## 🖼️ 功能展示

| 功能特点 | 界面展示 |
|----------|----------|
| **现代化界面** | 简洁美观的用户界面，支持拖拽操作 |
| **批量处理** | 一次性处理多个PDF文件和ZIP压缩包 |
| **实时进度** | 显示处理进度和详细日志信息 |
| **智能布局** | 2列4行拼版，节省50%纸张 |

### 主要界面功能
- 🎯 **文件选择**: 支持拖拽添加PDF文件和ZIP压缩包
- 📁 **批量处理**: 一次性处理多个发票文件  
- ⚡ **实时进度**: 显示处理进度和详细日志
- 🎨 **现代设计**: 简洁美观的用户界面
```

## 🔧 技术实现

```

https://raw.githubusercontent.com/yingcaihuang/invoice-pretty/main/img/assets/img2.png
```

- **`.github/workflows/build-and-release.yml`** - GitHub Actions工作流配置

- **`img/assets/img1.png`** (38.6 KB) - 系统主界面截图

## 🔄 更新维护

### 何时更新截图
1. **界面重大改版** - UI设计有显著变化
2. **功能新增** - 添加了重要的新功能
3. **用户反馈** - 用户建议更新展示内容

### 更新步骤
1. 替换 `img/assets/` 目录中的图片文件
2. 保持文件名不变 (`img1.png`, `img2.png`)
3. 推送到main分支
4. 下次发布时自动使用新截图

## 💡 最佳实践

### 截图质量
- **分辨率**: 建议使用高分辨率截图
- **文件大小**: 控制在100KB以内，确保加载速度
- **内容**: 展示软件的核心功能和界面

### Release Notes
- **结构清晰**: 使用标题和列表组织内容
- **信息完整**: 包含下载、安装、使用的完整指导
- **视觉友好**: 使用emoji和表格增强可读性
